<p>A large <a href="/wiki/index.php/Equilateral_triangle" title="Equilateral triangle">equilateral triangle</a> is constructed by using toothpicks to create rows of small equilateral triangles. For example, in the figure, we have <img src="//latex.artofproblemsolving.com/7/c/d/7cde695f2e4542fd01f860a89189f47a27143b66.png" class="latex" alt="$3$" width="8" height="12" /> rows of small congruent equilateral triangles, with <img src="//latex.artofproblemsolving.com/7/9/0/79069377f91364c2f87a64e5f9f562a091c8a6c1.png" class="latex" alt="$5$" width="8" height="12" /> small triangles in the base row.  How many toothpicks would be needed to construct a large equilateral triangle if the base row of the triangle consists of <img src="//latex.artofproblemsolving.com/b/9/1/b910f4dafca80b54c6e7bedf63b6485eebff0764.png" class="latex" alt="$2003$" width="35" height="12" /> small equilateral triangles? 
</p><p><img src="//latex.artofproblemsolving.com/1/6/c/16ca1f0305d713b2b9e5a0c53658d11829856ec7.png" class="latexcenter" alt="[asy] unitsize(15mm); defaultpen(linewidth(.8pt)+fontsize(8pt)); pair Ap=(0,0), Bp=(1,0), Cp=(2,0), Dp=(3,0), Gp=dir(60); pair Fp=shift(Gp)*Bp, Ep=shift(Gp)*Cp; pair Hp=shift(Gp)*Gp, Ip=shift(Gp)*Fp; pair Jp=shift(Gp)*Hp; pair[] points={Ap,Bp,Cp,Dp,Ep,Fp,Gp,Hp,Ip,Jp}; draw(Ap--Dp--Jp--cycle); draw(Gp--Bp--Ip--Hp--Cp--Ep--cycle); for(pair p&#160;: points) { fill(circle(p, 0.07),white); } pair[] Cn=new pair[5]; Cn[0]=centroid(Ap,Bp,Gp); Cn[1]=centroid(Gp,Bp,Fp); Cn[2]=centroid(Bp,Fp,Cp); Cn[3]=centroid(Cp,Fp,Ep); Cn[4]=centroid(Cp,Ep,Dp); label(&quot;$1$&quot;,Cn[0]); label(&quot;$2$&quot;,Cn[1]); label(&quot;$3$&quot;,Cn[2]); label(&quot;$4$&quot;,Cn[3]); label(&quot;$5$&quot;,Cn[4]); for (pair p&#160;: Cn) { draw(circle(p,0.1)); }[/asy]" width="225" height="195" />
</p><p><img src="//latex.artofproblemsolving.com/0/8/3/0837bddd8e8fdd5cecc04a04167021c986d2d342.png" class="latex" alt="$\mathrm{(A) \ } 1,004,004 \qquad \mathrm{(B) \ } 1,005,006 \qquad \mathrm{(C) \ } 1,507,509 \qquad \mathrm{(D) \ } 3,015,018 \qquad \mathrm{(E) \ } 6,021,018$" style="vertical-align: -4px" width="695" height="18" />
</p><p><br />
</p>
<h2><span class="mw-headline" id="Solution_1">Solution 1</span></h2>
<p>There are <img src="//latex.artofproblemsolving.com/f/1/3/f13ba9f93ef56169929553418383922dafb68bb6.png" class="latex" alt="$1+3+5+...+2003=1002^{2}=1004004$" style="vertical-align: -1px" width="323" height="16" /> small equilateral triangles. 
</p><p>Each small equilateral triangle needs <img src="//latex.artofproblemsolving.com/7/c/d/7cde695f2e4542fd01f860a89189f47a27143b66.png" class="latex" alt="$3$" width="8" height="12" /> toothpicks to make it. 
</p><p>But, each toothpick that isn't one of the <img src="//latex.artofproblemsolving.com/2/4/d/24df3828d1ba088836fae8c9eba1968ac5a58ef4.png" class="latex" alt="$1002\cdot3=3006$" style="vertical-align: 0px" width="118" height="13" /> toothpicks on the outside of the large equilateral triangle is a side for <img src="//latex.artofproblemsolving.com/4/1/c/41c544263a265ff15498ee45f7392c5f86c6d151.png" class="latex" alt="$2$" width="8" height="12" /> small equilateral triangles. 
</p><p>So, the number of toothpicks on the inside of the large equilateral triangle is <img src="//latex.artofproblemsolving.com/2/f/0/2f05ccc355706da61af4ff719c4bdc1d272a5efb.png" class="latex" alt="$\frac{10040004\cdot3-3006}{2}=1504503$" style="vertical-align: -12px" width="244" height="37" />
</p><p>Therefore the total number of toothpicks is <img src="//latex.artofproblemsolving.com/d/0/c/d0c98d4d0572c24886c62b7f469b22b619a542ea.png" class="latex" alt="$1504503+3006=\boxed{\mathrm{(C)}\ 1,507,509}$" style="vertical-align: -10px" width="270" height="29" /> 
~dolphin7
</p>
<h2><span class="mw-headline" id="Solution_2">Solution 2</span></h2>
<p>We just need to count upward facing triangles because if we exclude the downward-facing triangles, we won't be overcounting any toothpicks. The first row of triangles has <img src="//latex.artofproblemsolving.com/d/c/e/dce34f4dfb2406144304ad0d6106c5382ddd1446.png" class="latex" alt="$1$" style="vertical-align: 0px" width="8" height="12" /> upward-facing triangle, the second row has <img src="//latex.artofproblemsolving.com/4/1/c/41c544263a265ff15498ee45f7392c5f86c6d151.png" class="latex" alt="$2$" width="8" height="12" /> upward-facing triangles, the third row has <img src="//latex.artofproblemsolving.com/7/c/d/7cde695f2e4542fd01f860a89189f47a27143b66.png" class="latex" alt="$3$" width="8" height="12" /> upward-facing triangles, and so on having <img src="//latex.artofproblemsolving.com/1/7/4/174fadd07fd54c9afe288e96558c92e0c1da733a.png" class="latex" alt="$n$" width="10" height="8" /> upward-facing triangles in the <img src="//latex.artofproblemsolving.com/3/6/5/3657b37b69a5e1029f9182897c6b5dc7492cc1bd.png" class="latex" alt="$n^\text{th}$" width="23" height="15" /> row. The last row with <img src="//latex.artofproblemsolving.com/b/9/1/b910f4dafca80b54c6e7bedf63b6485eebff0764.png" class="latex" alt="$2003$" width="35" height="12" /> small triangles has <img src="//latex.artofproblemsolving.com/6/3/a/63a8720f439e8e8887a0746430412c323e81e13c.png" class="latex" alt="$1002$" style="vertical-align: 0px" width="35" height="13" /> upward-facing triangles. By Gauss's formula, the number of the upward-facing triangles in the entire triangle are now <img src="//latex.artofproblemsolving.com/f/e/7/fe7c4e0f55a5db118407be24b20fdcfc42a2b73c.png" class="latex" alt="$\frac{1002\times1003}{2}$" style="vertical-align: -12px" width="97" height="37" />, meaning that the number of toothpicks are <img src="//latex.artofproblemsolving.com/a/d/a/ada473f8b019d281093f37afa617b1aedbebcf13.png" class="latex" alt="$\frac{1002\times1003}{2}\times3$" style="vertical-align: -12px" width="129" height="37" />, or <img src="//latex.artofproblemsolving.com/8/2/b/82b317654f901a9c8f56585721a617396d3b050c.png" class="latex" alt="$\boxed{\text{C}}$" style="vertical-align: -5px" width="23" height="23" />.
</p><p>~mathpro12345
</p>
<h3><span class="mw-headline" id="Note">Note</span></h3>
<p>You don't have to calculate the value of <img src="//latex.artofproblemsolving.com/a/d/a/ada473f8b019d281093f37afa617b1aedbebcf13.png" class="latex" alt="$\frac{1002\times1003}{2}\times3$" style="vertical-align: -12px" width="129" height="37" />, and you can use units digits to find the answer easily. The units digit of <img src="//latex.artofproblemsolving.com/8/a/6/8a6554a8e06e479445d02be2e279a19724f06955.png" class="latex" alt="$1002\times1003$" style="vertical-align: 0px" width="94" height="13" /> is <img src="//latex.artofproblemsolving.com/6/0/1/601a7806cbfad68196c43a4665871f8c3186802e.png" class="latex" alt="$6$" width="8" height="12" />, and has a unit digit of <img src="//latex.artofproblemsolving.com/7/c/d/7cde695f2e4542fd01f860a89189f47a27143b66.png" class="latex" alt="$3$" width="8" height="12" /> after being divided by <img src="//latex.artofproblemsolving.com/4/1/c/41c544263a265ff15498ee45f7392c5f86c6d151.png" class="latex" alt="$2$" width="8" height="12" />. Then this is multiplied by <img src="//latex.artofproblemsolving.com/7/c/d/7cde695f2e4542fd01f860a89189f47a27143b66.png" class="latex" alt="$3$" width="8" height="12" />, now the final number ending with a <img src="//latex.artofproblemsolving.com/b/f/2/bf2c9074b396e3af0dea52d792660eea1c77f10f.png" class="latex" alt="$9$" width="8" height="12" />. This leaves only one answer choice possible, which is <img src="//latex.artofproblemsolving.com/8/2/b/82b317654f901a9c8f56585721a617396d3b050c.png" class="latex" alt="$\boxed{\text{C}}$" style="vertical-align: -5px" width="23" height="23" />
</p><p><br />
</p>
<h3><span class="mw-headline" id="Solution_3">Solution 3</span></h3>
<p>Test out some fewer cases first.
</p><p>When there is just 1 equilateral triangle in the base, you need <img src="//latex.artofproblemsolving.com/7/c/d/7cde695f2e4542fd01f860a89189f47a27143b66.png" class="latex" alt="$3$" width="8" height="12" /> toothpicks.
When there are 3 equilateral triangles in the base, you need <img src="//latex.artofproblemsolving.com/b/f/2/bf2c9074b396e3af0dea52d792660eea1c77f10f.png" class="latex" alt="$9$" width="8" height="12" /> toothpicks in all.
When there are 5 equilateral triangles in the base, you need <img src="//latex.artofproblemsolving.com/5/e/7/5e7a1b6098a8c98b8e4adae526aeef4b91712620.png" class="latex" alt="$18$" style="vertical-align: 0px" width="17" height="13" /> toothpicks in all.
When there are 7 equilateral triangles in the base, you need <img src="//latex.artofproblemsolving.com/6/5/1/651ba418a810d971dbc8a326d49a71fe1541fae0.png" class="latex" alt="$30$" width="17" height="12" /> toothpicks in all.
</p><p>Taking the finite differences, we get <img src="//latex.artofproblemsolving.com/2/5/a/25a8e7a658123a76226358a4a1c6343ab33d3f8b.png" class="latex" alt="$6, 9, 12.$" style="vertical-align: -3px" width="56" height="16" /> It forms a linear equations. This means the original numbers <img src="//latex.artofproblemsolving.com/c/b/1/cb1813c2fec77f4517e463e6d86a50d9be8f64b3.png" class="latex" alt="$(3, 9, 18, 30)$" style="vertical-align: -4px" width="91" height="18" /> form a quadratic.
</p><p>Let the quadratic be <img src="//latex.artofproblemsolving.com/8/7/8/878740b1e9bad0008122348eca48d8883a00e7ea.png" class="latex" alt="$y = ax^2 + bx + c$" style="vertical-align: -3px" width="132" height="18" /> where <img src="//latex.artofproblemsolving.com/a/3/5/a352c940a60100065e251209e4689399b96dfe8e.png" class="latex" alt="$y = 2* \text{equilateral triangles in base} - 1.$" style="vertical-align: -3px" width="295" height="15" />
</p><p>Then, we have the following points: <img src="//latex.artofproblemsolving.com/3/b/c/3bc288a276f8bb20c1cc49075b8fdf681d9f1ce8.png" class="latex" alt="$(1, 3), (2, 9), (3, 18), (4, 30).$" style="vertical-align: -4px" width="207" height="18" />
</p><p>We can plug these values into <img src="//latex.artofproblemsolving.com/8/7/8/878740b1e9bad0008122348eca48d8883a00e7ea.png" class="latex" alt="$y = ax^2 + bx + c$" style="vertical-align: -3px" width="132" height="18" />, giving: 
</p><p><img src="//latex.artofproblemsolving.com/6/8/d/68d63eda80865471ce09ec436a36cf325a35b1e5.png" class="latexcenter" alt="\[a + b + c = 3, 4a + b + c = 9, 9a + 3b + c = 18.\]" width="364" height="16" />
</p><p>Solving gives <img src="//latex.artofproblemsolving.com/7/f/d/7fd292131d0718f0700bac0d29566affb59f2b62.png" class="latex" alt="$a = b = 1.5, c = 0.$" style="vertical-align: -3px" width="142" height="16" /> So, <img src="//latex.artofproblemsolving.com/3/c/7/3c7e95ae7c302c00ea33805957ed6c489e08014d.png" class="latexcenter" alt="\[y = 1.5x^2 + 1.5x.\]" width="133" height="18" />
</p><p>For our problem, we need it when there are <img src="//latex.artofproblemsolving.com/b/9/1/b910f4dafca80b54c6e7bedf63b6485eebff0764.png" class="latex" alt="$2003$" width="35" height="12" /> equilateral triangles in the base. For the quadratic, the corresponding <img src="//latex.artofproblemsolving.com/2/6/e/26eeb5258ca5099acf8fe96b2a1049c48c89a5e6.png" class="latex" alt="$x$" width="10" height="8" />-value would be <img src="//latex.artofproblemsolving.com/a/1/a/a1acaa0ba86d1202033443a5b9cc05afd7d22f71.png" class="latex" alt="$\frac{2003 + 1}{2} = 1002.$" style="vertical-align: -12px" width="135" height="37" />. So, our answer is simply: <img src="//latex.artofproblemsolving.com/1/c/d/1cd41da3ca58645022d776c8b615b657276b64d7.png" class="latexcenter" alt="\[1.5 * 1002^2 + 1.5*1002 = \boxed{1507509}.\]" width="284" height="23" />
</p><p><br />
</p>
