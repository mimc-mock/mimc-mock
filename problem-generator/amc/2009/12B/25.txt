<h2><span class="mw-headline" id="Solution">Solution</span></h2>
<p>We need to find a reasonably easy way to count the squares. 
</p><p>First, obviously the maximum distance between two points in the same quadrant is <img src="//latex.artofproblemsolving.com/4/a/0/4a07486561371ba2da71de5d0e18dac87884de00.png" class="latex" alt="$4\sqrt 2 &lt; 6$" style="vertical-align: -1px" width="66" height="18" />, hence each square has exactly one vertex in each quadrant.
</p><p>Given any square, we can circumscribe another axes-parallel square around it. In the picture below, the original square is red and the circumscribed one is blue.
</p><p><img src="//latex.artofproblemsolving.com/d/6/e/d6ea5bf041c819b0aac88605e9ce17944ad62acf.png" class="latexcenter" alt="[asy] defaultpen(black+0.75bp+fontsize(8pt)); size(7.5cm); path p = scale(.15)*unitcircle; draw((-8,0)--(8.5,0),Arrow(HookHead,1mm)); draw((0,-8)--(0,8.5),Arrow(HookHead,1mm)); int i,j; for (i=-7;i&lt;8;++i) { for (j=-7;j&lt;8;++j) { if (((-7 &lt;= i) &amp;&amp; (i &lt;= -3)) || ((3 &lt;= i) &amp;&amp;  (i&lt;= 7))) { if (((-7 &lt;= j) &amp;&amp; (j &lt;= -3)) || ((3 &lt;= j) &amp;&amp;  (j&lt;= 7))) { fill(shift(i,j)*p,black); }}}} draw((-7,-.2)--(-7,.2),black+0.5bp); draw((-3,-.2)--(-3,.2),black+0.5bp); draw((3,-.2)--(3,.2),black+0.5bp); draw((7,-.2)--(7,.2),black+0.5bp); draw((-.2,-7)--(.2,-7),black+0.5bp); draw((-.2,-3)--(.2,-3),black+0.5bp); draw((-.2,3)--(.2,3),black+0.5bp); draw((-.2,7)--(.2,7),black+0.5bp); label(&quot;$-7$&quot;,(-7,0),S); label(&quot;$-3$&quot;,(-3,0),S); label(&quot;$3$&quot;,(3,0),S); label(&quot;$7$&quot;,(7,0),S); label(&quot;$-7$&quot;,(0,-7),W); label(&quot;$-3$&quot;,(0,-3),W); label(&quot;$3$&quot;,(0,3),W); label(&quot;$7$&quot;,(0,7),W); draw( (5,3) -- (-3,4) -- (-4,-4) -- (4,-5) -- cycle, red ); draw( (5,4) -- (-4,4) -- (-4,-5) -- (5,-5) -- cycle, dashed + blue ); [/asy]" width="355" height="355" />
</p><p>Let's now consider the opposite direction. Assume that we picked the blue square, how many different red squares do share it?
</p><p>Answering this question is not as simple as it may seem. Consider the picture below. It shows all three red squares that share the same blue square. In addition, the picture shows a green square that is not valid, as two of its vertices are in bad locations.
</p><p><img src="//latex.artofproblemsolving.com/5/f/e/5fe3171c99e955ea90e3e3c28e14eb23886c1904.png" class="latexcenter" alt="[asy] defaultpen(black+0.75bp+fontsize(8pt)); size(7.5cm); path p = scale(.15)*unitcircle; draw((-8,0)--(8.5,0),Arrow(HookHead,1mm)); draw((0,-8)--(0,8.5),Arrow(HookHead,1mm)); int i,j; for (i=-7;i&lt;8;++i) { for (j=-7;j&lt;8;++j) { if (((-7 &lt;= i) &amp;&amp; (i &lt;= -3)) || ((3 &lt;= i) &amp;&amp;  (i&lt;= 7))) { if (((-7 &lt;= j) &amp;&amp; (j &lt;= -3)) || ((3 &lt;= j) &amp;&amp;  (j&lt;= 7))) { fill(shift(i,j)*p,black); }}}} draw((-7,-.2)--(-7,.2),black+0.5bp); draw((-3,-.2)--(-3,.2),black+0.5bp); draw((3,-.2)--(3,.2),black+0.5bp); draw((7,-.2)--(7,.2),black+0.5bp); draw((-.2,-7)--(.2,-7),black+0.5bp); draw((-.2,-3)--(.2,-3),black+0.5bp); draw((-.2,3)--(.2,3),black+0.5bp); draw((-.2,7)--(.2,7),black+0.5bp); label(&quot;$-7$&quot;,(-7,0),S); label(&quot;$-3$&quot;,(-3,0),S); label(&quot;$3$&quot;,(3,0),S); label(&quot;$7$&quot;,(7,0),S); label(&quot;$-7$&quot;,(0,-7),W); label(&quot;$-3$&quot;,(0,-3),W); label(&quot;$3$&quot;,(0,3),W); label(&quot;$7$&quot;,(0,7),W); draw( (5,4) -- (-4,4) -- (-4,-5) -- (5,-5) -- cycle, red ); draw( (5,3) -- (-3,4) -- (-4,-4) -- (4,-5) -- cycle, red ); draw( (5,-3) -- (-2,-5) -- (-4,2) -- (3,4) -- cycle, dashed + green ); draw( (5,-4) -- (-3,-5) -- (-4,3) -- (4,4) -- cycle, red ); draw( scale(1.05)*((5,4) -- (-4,4) -- (-4,-5) -- (5,-5) -- cycle), dashed + blue ); [/asy]" width="355" height="355" />

<h2><span class="mw-headline" id="Solution_2">Solution 2</span></h2>
<p>This is based on a clever bijection given in <a rel="nofollow" class="external text" href="http://web2.slc.qc.ca/sh/Contest/AMC12_2009B-S.pdf">this page</a>.
</p><p>Consider any square <img src="//latex.artofproblemsolving.com/f/9/e/f9efaf9474c5658c4089523e2aff4e11488f8603.png" class="latex" alt="$ABCD$" width="57" height="13" /> where all four vertices are in <img src="//latex.artofproblemsolving.com/6/e/2/6e28ce12d49d39f160d5a0ef54077fc98e4b9d2b.png" class="latex" alt="$G$" width="14" height="12" />, and the side length is at least <img src="//latex.artofproblemsolving.com/6/0/1/601a7806cbfad68196c43a4665871f8c3186802e.png" class="latex" alt="$6$" width="8" height="12" />, so the four vertices must lie in distinct quadrants (Same proof as in solution 1). Without loss of generality, assume that <img src="//latex.artofproblemsolving.com/8/8/5/8859dbfb7a9ec734e16f2a88fb5cf1104eb6a39b.png" class="latex" alt="$A,B,C,D$" style="vertical-align: -3px" width="81" height="16" /> are in the first, second, third, fourth quadrant. Then we consider the following mapping:
</p><p><img src="//latex.artofproblemsolving.com/a/3/5/a353773d3b0451b8e94d66eb094013e417c2a11f.png" class="latexcenter" alt="\[A \to A&#39; = A\]" width="97" height="14" />
<img src="//latex.artofproblemsolving.com/6/5/d/65dc39ccde442f164ea4ea5884b47228801b5708.png" class="latexcenter" alt="\[B \to B&#39; = B + (10,0)\]" width="171" height="18" />
<img src="//latex.artofproblemsolving.com/1/f/3/1f3554abd2ef7369621ed490648b984e8b289daf.png" class="latexcenter" alt="\[C \to C&#39; = C + (10,10)\]" width="178" height="18" />
<img src="//latex.artofproblemsolving.com/a/5/8/a585bffd4daf45f3445b7eb8c92a21f7b34ec1b8.png" class="latexcenter" alt="\[D \to D&#39; = D + (0,10)\]" width="173" height="18" />
</p><p>Then the new points <img src="//latex.artofproblemsolving.com/1/0/9/1090530ae883b627e090bfb5e450ef796febca40.png" class="latex" alt="$A&#39;$" width="17" height="14" />, <img src="//latex.artofproblemsolving.com/4/c/7/4c71f19552a05ee5e2b31e94aff3fb986884bf31.png" class="latex" alt="$B&#39;$" width="18" height="14" />, <img src="//latex.artofproblemsolving.com/9/b/7/9b798193cb94562989018728e1151587c618655b.png" class="latex" alt="$C&#39;$" style="vertical-align: -1px" width="21" height="16" />, <img src="//latex.artofproblemsolving.com/b/2/3/b239e47c75fa1adfb0e7d7fb6bf219fcc53f86d9.png" class="latex" alt="$D&#39;$" width="19" height="14" /> are either being the same point or forming a square in <img src="//latex.artofproblemsolving.com/4/7/4/4747455c8ade5bc4fa20306ddd76ed675178c499.png" class="latex" alt="$G_1 = G \cap \{ x&gt;0, y&gt;0 \}$" style="vertical-align: -4px" width="192" height="18" />, a 5x5 grid. 
</p><p>Conversely, for any point in <img src="//latex.artofproblemsolving.com/8/d/b/8db8d2c3b6325306a87e8a10b0e5358afb2f23eb.png" class="latex" alt="$G_1$" style="vertical-align: -2px" width="20" height="15" />, it can be reversed to a square <img src="//latex.artofproblemsolving.com/f/9/e/f9efaf9474c5658c4089523e2aff4e11488f8603.png" class="latex" alt="$ABCD$" width="57" height="13" />; however, for any square in <img src="//latex.artofproblemsolving.com/8/d/b/8db8d2c3b6325306a87e8a10b0e5358afb2f23eb.png" class="latex" alt="$G_1$" style="vertical-align: -2px" width="20" height="15" />, there are four possible squares <img src="//latex.artofproblemsolving.com/f/9/e/f9efaf9474c5658c4089523e2aff4e11488f8603.png" class="latex" alt="$ABCD$" width="57" height="13" /> that were mapped to them. Therefore the number of possible squares <img src="//latex.artofproblemsolving.com/f/9/e/f9efaf9474c5658c4089523e2aff4e11488f8603.png" class="latex" alt="$ABCD$" width="57" height="13" /> is equal to <img src="//latex.artofproblemsolving.com/8/a/2/8a20bbdbcedd6384e88d116cb4cf581baa6ce124.png" class="latex" alt="$25 + 4N$" style="vertical-align: -1px" width="66" height="14" />, where <img src="//latex.artofproblemsolving.com/f/c/9/fc97ef67268cd4e91bacdf12b8901d7036c9a056.png" class="latex" alt="$N$" width="16" height="12" /> is the number of squares inscribed in <img src="//latex.artofproblemsolving.com/8/d/b/8db8d2c3b6325306a87e8a10b0e5358afb2f23eb.png" class="latex" alt="$G_1$" style="vertical-align: -2px" width="20" height="15" />.
</p><p>Moreover by the same idea in solution 1, each square (with sides parallel or slanted to the axes) in a <img src="//latex.artofproblemsolving.com/8/d/b/8db8d2c3b6325306a87e8a10b0e5358afb2f23eb.png" class="latex" alt="$G_1$" style="vertical-align: -2px" width="20" height="15" /> can be inscribed in a square in <img src="//latex.artofproblemsolving.com/8/d/b/8db8d2c3b6325306a87e8a10b0e5358afb2f23eb.png" class="latex" alt="$G_1$" style="vertical-align: -2px" width="20" height="15" />, with sides parallel to one of the axes, call it "standard square". Noticing that each standard square of side length <img src="//latex.artofproblemsolving.com/c/7/d/c7d457e388298246adb06c587bccd419ea67f7e8.png" class="latex" alt="$a$" width="9" height="8" /> corresponds to <img src="//latex.artofproblemsolving.com/c/7/d/c7d457e388298246adb06c587bccd419ea67f7e8.png" class="latex" alt="$a$" width="9" height="8" /> inscribed squares,  and that there are <img src="//latex.artofproblemsolving.com/0/a/b/0ab2b19209f865a1d24c23db1dc1e3f7ebcbbab9.png" class="latex" alt="$(5-a)^2$" style="vertical-align: -4px" width="61" height="19" /> number of standard squares of side length <img src="//latex.artofproblemsolving.com/c/7/d/c7d457e388298246adb06c587bccd419ea67f7e8.png" class="latex" alt="$a$" width="9" height="8" />, we have
</p><p><img src="//latex.artofproblemsolving.com/9/b/0/9b05cb3509ccac175176d88e249e9f86963e2392.png" class="latexcenter" alt="\[N = \sum_{a=1}^{4} a(5-a)^2 = 1\cdot 16 + 2\cdot 9 + 3 \cdot 4 + 4 \cdot 1 = 16+18+12+4=50\]" width="479" height="50" />
</p><p>So the answer is <img src="//latex.artofproblemsolving.com/2/c/6/2c6fe9ad4b1722fa4fbb6c36b13caa74a7c62fde.png" class="latex" alt="$25+4\cdot 50 = 225$" style="vertical-align: -1px" width="131" height="14" />
