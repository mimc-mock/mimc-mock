We use Legendre's Formula and split up into two cases:
</p><p><img src="//latex.artofproblemsolving.com/c/c/a/ccad89fc6bf3acedebe15f57cf963968ea762596.png" class="latex" alt="$\textbf{CASE ONE: } 5\leq 2n &lt; 25.$" style="vertical-align: -3px" width="214" height="16" />
</p><p>The only way we can fulfill the requirements is if <img src="//latex.artofproblemsolving.com/0/3/4/034315345e79187324657032323d21e19b4ae5d9.png" class="latex" alt="$\lfloor{\dfrac{n}{5}}\rfloor = 1$" style="vertical-align: -13px" width="66" height="34" /> and <img src="//latex.artofproblemsolving.com/e/b/8/eb8a0626279afe534862e4486ceeb801849bfc9f.png" class="latex" alt="$\lfloor{\dfrac{2n}{5}}\rfloor=3$" style="vertical-align: -13px" width="75" height="38" /> which means that <img src="//latex.artofproblemsolving.com/4/7/b/47b250d1dcec253dd9d80d69d48f7c9a6bbdc82b.png" class="latex" alt="$5\leq n &lt;10$" style="vertical-align: -3px" width="89" height="16" /> and <img src="//latex.artofproblemsolving.com/2/0/f/20f33bd8746dcf5762f8ef1138c5f005e2ee620d.png" class="latex" alt="$15\leq 2n &lt; 20$" style="vertical-align: -3px" width="107" height="16" />. The only way this works is if <img src="//latex.artofproblemsolving.com/c/e/0/ce095676fc619c1b76a6beb63c199bf1ef5999c7.png" class="latex" alt="$n = 8 \text{ or } 9.$" style="vertical-align: -1px" width="86" height="14" />
</p><p><img src="//latex.artofproblemsolving.com/4/7/e/47ed51422fa76e2be664156545a4ce60946c491f.png" class="latex" alt="$\textbf{CASE TWO: } 25 \leq 2n$" style="vertical-align: -3px" width="181" height="16" />.
</p><p>Since we want the smallest values of <img src="//latex.artofproblemsolving.com/1/7/4/174fadd07fd54c9afe288e96558c92e0c1da733a.png" class="latex" alt="$n$" width="10" height="8" />, we first try it when <img src="//latex.artofproblemsolving.com/8/2/a/82a3ef1aae843b6dfbfa0f1d722ac24999e8d125.png" class="latex" alt="$2n&lt;30.$" style="vertical-align: -2px" width="69" height="15" /> Thus <img src="//latex.artofproblemsolving.com/e/7/c/e7c65c2a874b19bdb25e07737a8a098ccd4c292e.png" class="latex" alt="$(2n)!$" style="vertical-align: -4px" width="37" height="18" /> has 6 zeros, which implies that <img src="//latex.artofproblemsolving.com/6/3/f/63f344e7d8b14a7886fc7fb72da53cd0a3271dcc.png" class="latex" alt="$n!$" width="14" height="13" /> must have 2. The only way to do this while maintaining our restrictions for <img src="//latex.artofproblemsolving.com/f/9/3/f93f5c51ea4b04b4992b003b39479f29018f6bda.png" class="latex" alt="$2n$" width="19" height="12" /> is if <img src="//latex.artofproblemsolving.com/b/2/4/b241d315e2f66fbe2488ae38db2d104c5ba6aa30.png" class="latex" alt="$n = 13 \text{ or } 14.$" style="vertical-align: -1px" width="104" height="14" />
</p><p>So the sum of the four values is <img src="//latex.artofproblemsolving.com/a/d/8/ad81124f99f73313ff91d68b3b04fbdddcaa97ef.png" class="latex" alt="$8+9+13+14=44$" style="vertical-align: -2px" width="165" height="15" /> so the digit is sum is <img src="//latex.artofproblemsolving.com/1/3/9/139aff62411176aa6c91cdbb56efa5dca341520b.png" class="latex" alt="$\boxed{\mathbf{(B)\ }8}.$" style="vertical-align: -11px" width="62" height="32" />
</p><p>-ConfidentKoala4
</p>
